MOC Project Intake Form

Complete all fields to submit your project request

Project Name

The Project Name should be concise, descriptive, and locally unique. You may be asked for a different name if a current project has a matching or confusingly similar name.

Project Overview

The Project Overview is one or two paragraphs that summarize what you want to do. Longer than an elevator pitch, shorter than a TED talk.

Project Administrative Information

Who wants resources? What is the source budget? Who should be notified about charges, projections, and other administrative information?

Project Timeline

When do you want to start, when do you plan to end? Any constraints?

Technical Requirements

Describe your technical requirements in detail. Consider: general description, runtime environment, hardware, networking, storage, and externalities.

User Accounts

This must contain a principal investigator (PI) that is responsible for deciding how their resources are used and communicating with the MOC. This may contain additional users that the PI would like to share their resources with for collaboration.

Additional Notes / Reading

Anything that would help the MOC understand the project is appreciated. This information may uncover technical edge cases that must be triaged before approval.

Attachments

Any diagrams or other material that may be required to understand the project.
